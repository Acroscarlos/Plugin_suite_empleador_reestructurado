### 1. VEREDICTO ARQUITECTÓNICO: Opción A vs Opción B

**Mi recomendación definitiva es la OPCIÓN B (Interceptor 2FA 100% Personalizado).**

*Justificación Técnica:* 
Aunque en un inicio integrar un plugin como *Wordfence* o *WP 2FA* (Opción A) parece la vía rápida para no "reinventar la rueda criptográfica", estos plugins están profundamente acoplados al flujo nativo de WordPress (`wp-login.php`, `wp-admin`, hooks `authenticate`). Como nuestro ERP `SuiteEmpleados` utiliza una arquitectura **100% AJAX/Headless** con modales en el frontend, intentar interceptar un plugin de terceros generaría un código "espagueti", frágil ante actualizaciones y con una pésima experiencia de usuario.

Para la **Opción B**, no escribiremos la criptografía desde cero (eso sería un riesgo), sino que integraremos una librería PHP probada y de un solo archivo (ej. `PHPGangsta_GoogleAuthenticator`) directamente en nuestra capa `includes/Core/`. Esto nos da control absoluto del flujo y una UI fluida con SweetAlert2.

---

### 2. ARQUITECTURA DE BASE DE DATOS Y ESTADO (Zero-Trust)

No confiaremos en el cliente durante el proceso. Guardaremos el estado intermedio en el servidor.
*   **User Meta Guardados:**
    *   `suite_2fa_secret` (String): La llave criptográfica base32.
    *   `suite_2fa_enabled` (Booleano/String '1' o '0'): Bandera de activación.
*   **Transients de WordPress:** 
    *   `suite_2fa_pending_{hash}`: Almacenará el ID del usuario temporalmente (por 5 minutos) tras validar su contraseña, pero *antes* de entregarle la cookie de sesión final.

---

### 3. DISEÑO DEL FLUJO EN EL FRONTEND (SPA / SweetAlert2)

1. **Paso 1:** El usuario ingresa Email y Contraseña en tu login actual. Se dispara AJAX a `suite_login_auth`.
2. **Paso 2:** Si las credenciales son válidas, el backend **NO** lo loguea. Devuelve un `hash` temporal y un estado (`verify` o `enroll`).
3. **Paso 3 (Enroll):** Si es su primera vez, SweetAlert2 muestra un código QR. El usuario lo escanea, ingresa el código de 6 dígitos en el modal y se envía a verificar.
4. **Paso 4 (Verify):** Si ya estaba enrolado, SweetAlert2 solo pide los 6 dígitos. Al verificar, el backend emite la cookie (`wp_set_auth_cookie`) y redirige al Dashboard.

---

### 4. CÓDIGO QUIRÚRGICO DE IMPLEMENTACIÓN

*Pre-requisito:* Descarga la clase `GoogleAuthenticator.php` (de PHPGangsta o similar) y colócala en `includes/Core/class-google-authenticator.php`.

#### BLOQUE A: Controladores de Login y Verificación (Backend)
**Archivo:** `includes/Controllers/Ajax/class-suite-ajax-login.php` (Crea este archivo e instáncialo en `suite-empleados.php`).

```php
<?php
if ( ! defined( 'ABSPATH' ) ) { exit; }

require_once SUITE_PATH . 'includes/Core/class-google-authenticator.php'; // Tu librería TOTP

/**
 * Paso 1: Autenticación de Credenciales (Sin emitir cookie)
 */
class Suite_Ajax_Login_Auth extends Suite_AJAX_Controller {
    protected $action_name = 'suite_login_auth';
    protected $required_capability = ''; // Público

    protected function process() {
        $email = sanitize_email( $_POST['email'] );
        $password = $_POST['password'];

        // Validar credenciales nativas
        $user = wp_authenticate( $email, $password );
        if ( is_wp_error( $user ) ) {
            $this->send_error( 'Credenciales incorrectas.', 401 );
        }

        // Crear token de sesión temporal (Zero-Trust: 5 minutos de validez)
        $temp_hash = wp_generate_password( 32, false );
        set_transient( 'suite_2fa_pending_' . $temp_hash, $user->ID, 300 );

        $is_2fa_enabled = get_user_meta( $user->ID, 'suite_2fa_enabled', true ) === '1';

        if ( $is_2fa_enabled ) {
            // Ya enrolado, pedir código
            $this->send_success([ 'step' => 'verify', 'hash' => $temp_hash ]);
        } else {
            // No enrolado, generar secreto y QR
            $ga = new PHPGangsta_GoogleAuthenticator();
            $secret = $ga->createSecret();
            update_user_meta( $user->ID, 'suite_2fa_temp_secret', $secret ); // Secreto no confirmado
            
            $qr_url = $ga->getQRCodeGoogleUrl( 'SuiteEmpleados (' . $user->user_email . ')', $secret );
            $this->send_success([ 
                'step' => 'enroll', 
                'hash' => $temp_hash, 
                'qr_url' => $qr_url, 
                'secret' => $secret 
            ]);
        }
    }
}

/**
 * Paso 2: Validación TOTP y Emisión de Cookie
 */
class Suite_Ajax_Verify_2FA extends Suite_AJAX_Controller {
    protected $action_name = 'suite_verify_2fa';
    protected $required_capability = ''; // Público

    protected function process() {
        $hash = sanitize_text_field( $_POST['hash'] );
        $code = sanitize_text_field( $_POST['code'] );

        $user_id = get_transient( 'suite_2fa_pending_' . $hash );
        if ( ! $user_id ) {
            $this->send_error( 'Su sesión temporal ha expirado. Inicie sesión nuevamente.', 401 );
        }

        $is_2fa_enabled = get_user_meta( $user_id, 'suite_2fa_enabled', true ) === '1';
        // Leer el secreto definitivo o el temporal según el estado
        $secret = $is_2fa_enabled ? get_user_meta( $user_id, 'suite_2fa_secret', true ) : get_user_meta( $user_id, 'suite_2fa_temp_secret', true );

        $ga = new PHPGangsta_GoogleAuthenticator();
        $check_result = $ga->verifyCode( $secret, $code, 2 ); // 2 = Tolerancia de tiempo (2 * 30 seg)

        if ( $check_result ) {
            // Si era enrolamiento, consolidamos la seguridad
            if ( ! $is_2fa_enabled ) {
                update_user_meta( $user_id, 'suite_2fa_secret', $secret );
                update_user_meta( $user_id, 'suite_2fa_enabled', '1' );
                delete_user_meta( $user_id, 'suite_2fa_temp_secret' );
            }

            // Emitimos la cookie oficial de WordPress
            wp_set_auth_cookie( $user_id, true );
            delete_transient( 'suite_2fa_pending_' . $hash ); // Destruir token temporal

            if ( function_exists('suite_record_log') ) {
                suite_record_log( 'login_2fa', "Acceso 2FA exitoso para User ID: {$user_id}" );
            }

            $this->send_success( 'Acceso concedido.' );
        } else {
            // Mitigación de fuerza bruta: Podrías contar intentos fallidos aquí
            $this->send_error( 'Código 2FA inválido.', 401 );
        }
    }
}
```

#### BLOQUE B: Lógica del Frontend (JavaScript)
**Archivo:** El archivo JS que controle tu vista de login actual (ej. `assets/js/modules/login.js`).

```javascript
// --- INICIO FASE 2FA: INTERFAZ SWEETALERT2 ---

const handleLoginFlow = function(email, password) {
    // 1. Enviar credenciales (Fase 1)
    SuiteAPI.post('suite_login_auth', { email: email, password: password })
        .then(res => {
            const data = res.data;
            if (data.step === 'verify') {
                prompt2FAVerify(data.hash);
            } else if (data.step === 'enroll') {
                prompt2FAEnroll(data.hash, data.qr_url, data.secret);
            }
        })
        .catch(err => {
            Swal.fire('Acceso Denegado', err.responseJSON?.data || 'Credenciales inválidas.', 'error');
        });
};

const prompt2FAVerify = function(hash) {
    Swal.fire({
        title: '🔐 Autenticación de Dos Pasos',
        text: 'Abra su app de autenticación (Google Authenticator/Authy) e ingrese el código de 6 dígitos.',
        input: 'text',
        inputAttributes: { maxlength: 6, pattern: '*', inputmode: 'numeric' },
        showCancelButton: true,
        confirmButtonText: 'Verificar y Entrar',
        confirmButtonColor: '#0f172a',
        preConfirm: (code) => {
            if (!code || code.length !== 6) Swal.showValidationMessage('Ingrese un código de 6 dígitos válido');
            return code;
        }
    }).then((result) => {
        if (result.isConfirmed) {
            submit2FA(hash, result.value);
        }
    });
};

const prompt2FAEnroll = function(hash, qrUrl, secretText) {
    Swal.fire({
        title: '📱 Vincula tu Dispositivo (Obligatorio)',
        html: `
            <p style="font-size:14px; color:#475569;">Es su primera vez iniciando sesión. Escanee este código QR con Google Authenticator o Authy para habilitar el 2FA.</p>
            <img src="${qrUrl}" alt="QR Code" style="margin: 15px auto; border-radius:8px; border:2px solid #e2e8f0;">
            <p style="font-size:12px; color:#94a3b8;">Clave manual: <strong>${secretText}</strong></p>
            <hr style="margin: 15px 0;">
            <p style="font-size:14px; font-weight:bold;">Ingrese el código generado para confirmar:</p>
            <input type="text" id="swal-2fa-enroll-code" class="swal2-input" maxlength="6" style="text-align:center; font-size:24px; letter-spacing:5px;">
        `,
        showCancelButton: true,
        confirmButtonText: 'Vincular y Entrar',
        confirmButtonColor: '#059669',
        preConfirm: () => {
            const code = document.getElementById('swal-2fa-enroll-code').value;
            if (!code || code.length !== 6) Swal.showValidationMessage('Ingrese el código de 6 dígitos.');
            return code;
        }
    }).then((result) => {
        if (result.isConfirmed) {
            submit2FA(hash, result.value);
        }
    });
};

const submit2FA = function(hash, code) {
    Swal.fire({ title: 'Verificando...', allowOutsideClick: false, didOpen: () => Swal.showLoading() });

    SuiteAPI.post('suite_verify_2fa', { hash: hash, code: code })
        .then(res => {
            Swal.fire({ icon: 'success', title: '¡Acceso Concedido!', showConfirmButton: false, timer: 1500 });
            // Redirigir a la Intranet de tu ERP
            setTimeout(() => { window.location.href = '/intranet_1'; }, 1500); 
        })
        .catch(err => {
            Swal.fire('Código Inválido', 'El código ha expirado o es incorrecto.', 'error').then(() => {
                prompt2FAVerify(hash); // Volver a pedir
            });
        });
};
```
